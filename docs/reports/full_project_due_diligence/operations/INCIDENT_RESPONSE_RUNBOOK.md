# Incident Response Runbook

1. Triage severity and affected data/workflows.
2. Preserve API, DB, auth provider, storage, and audit logs.
3. Contain: revoke sessions/tokens, suspend users, disable public links if needed.
4. Fix and deploy through tested path.
5. Validate reporting integrity and approved-only data.
6. Notify stakeholders according to legal/privacy obligations.
7. Record post-incident actions and follow-up controls.
